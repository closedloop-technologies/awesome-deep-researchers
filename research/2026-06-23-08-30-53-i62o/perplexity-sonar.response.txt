### 1. **Top deep‑research APIs and providers (2026)**

Prioritize provider docs, pricing tiers, and supported patterns (agentic, multi‑step, structured output, file‑search, etc.).

#### 1.1 **Google Gemini Deep Research (Interactions API)**

- **Type:** Agentic, multi‑step researcher with reasoning, planning, iteration, and citation.
- **Docs:**
  - Overview: [Gemini Deep Research Agent](https://ai.google.dev/gemini-api/docs/interactions/deep-research) (Google AI for Developers)
  - Deep Research preview: [Deep Research preview](https://ai.google.dev/gemini-api/docs/models/deep-research-preview-04-2026)
- **Key features:**
  - End‑to‑end autonomous research: plan → search → read → iterate → output.
  - Multi‑file input and file search (your own docs, sheets, PDFs).
  - Visualization support (charts, graphs alongside reports).
  - Integration with MCP servers and external tools.
  - Two modes: “Deep Research” (fast) and “Deep Research Max” (maximum comprehensiveness).
- **Delivery:**
  - Via Google AI Studio (free tier) and Vertex AI (paid).
  - Enterprise access via Google Cloud partners.
- **Use cases:**
  - Market analysis, due diligence, competitive landscaping, literature reviews, long‑form reports.
- **Benchmark angle:**
  - Official blog: [Deep Research Max: a step change for autonomous research agents](https://blog.google/innovation-and-ai/models-and-research/gemini-models/next-generation-gemini-deep-research/)
- **Actionable for repo:**
  - Add a “Gemini Deep Research” section with `docs`, `quickstart`, `parameter tuning`, and `file‑search patterns`.
  - Include a minimal “research plan → report” template using Interactions API.

#### 1.2 **OpenAI Deep Research (ChatGPT Deep Research agent)**

- **Type:** Agentic, multi‑step web‑based researcher bundled into ChatGPT.
- **Docs:**
  - Announce + conceptual overview: [Introducing deep research – OpenAI](https://openai.com/index/introducing-deep-research/)
- **Key features:**
  - Accepts prompts and runs multi‑step web research, analysis, synthesis, and pivoting.
  - Supports file attachments (spreadsheets, PDFs) for context.
  - Auto‑generates step‑by‑step work logs and cites sources.
  - Engineered around a version of the upcoming `o3` model optimized for web browsing and data analysis.
- **Current access:**
  - Pro tier (100 queries/month starting point), followed by Plus/Team and Enterprise.
  - Delivered via ChatGPT web; mobile/desktop rollout “within the month” as of Feb 2026.
- **Emerging pattern:**
  - Connects to MCPs and “trusted‑site‑restricted” web scopes for enterprise use.
- **Actionable for repo:**
  - Create an “OpenAI Deep Research” pattern folder with:
    - `cost‑per‑query` estimates,
    - `query‑structs` (e.g., “compare X vs Y vs Z, output comparison table”),
    - `citation‑handling` and `evidence‑grading` macros.

#### 1.3 **Perplexity Sonar API (Deep Research Experience)**

- **Type:** Conversational, citation‑heavy deep research.
- **Docs / Info:**
  - Official API: via [Perplexity platform](https://www.perplexity.ai/); developer docs under API portal (requires account).
  - Product‑level context: [Perplexity as AI‑powered deep research](https://www.firecrawl.dev/blog/best-deep-research-apis) (Firecrawl blog, but cites Perplexity’s positioning).
- **Capabilities:**
  - Conversational synthesis from multiple web sources, with inline citations.
  - Used heavily for “scouting” and quick‑turn report‑style summaries.
- **Patterns:**
  - Great for “discovery first, verify later” pipelines.
- **Use in repo:**
  - Add a “Perplexity‑driven discovery” pattern, emphasizing:
    - prompt shapes for “generate source‑rich overview”,
    - post‑process citation‑filtering and reliability‑grading via a verifier model.

### 2. **Web‑focused and “deep‑crawl” research APIs**

These are not “fully agentic” researchers, but form the backbone of custom deep‑research stacks (scraping, search, content‑extraction).

#### 2.1 **Firecrawl**

- **Docs:**
  - Product + API: [Firecrawl](https://www.firecrawl.dev/)
  - Deep‑research API comparison: [Best Deep Research APIs for Agentic Workflows in 2026](https://www.firecrawl.dev/blog/best-deep-research-apis)
- **Capabilities:**
  - Autonomous web research via Agent endpoint.
  - Native Pydantic/Zod schema‑based extraction from crawled pages.
  - Flat‑rate (credit‑based) pricing; good for pipelines and RAG.
- **Positioning:**
  - Best when you want structured output, budget control, and autonomous research.
- **Actionable for repo:**
  - Pattern: “Firecrawl + OpenAI/Gemini” – scrape → extract → reason → synthesize.
  - Example: extract competitor landing pages → normalize into schema → run competitive memo.

#### 2.2 **Tavily**

- **Docs:**
  - API: [Tavily](https://tavily.com/)
  - Comparison: [Firecrawl blog on Tavily](https://www.firecrawl.dev/blog/best-deep-research-apis)
- **Capabilities:**
  - Query → JSON response with search results and snippets.
  - Fast, AI‑ready output; not an autonomous agent per se.
- **Best for:**
  - Search grounding in existing LLM apps (LangChain, LlamaIndex plugins).
- **Actionable for repo:**
  - Add a “Tavily ground‑then‑think” pattern: use Tavily to fetch raw snippets, then dispatch to a reasoning model for summarization.

#### 2.3 **Exa**

- **Docs:**
  - API: [Exa](https://exa.ai/)
  - Evaluation: [Firecrawl blog on Exa](https://www.firecrawl.dev/blog/best-deep-research-apis)
- **Capabilities:**
  - Neural‑semantic search and “Find Similar” for content.
  - Basic JSON result structures.
  - Great for semantic discovery, topic‑space exploration, and knowledge graphs.
- **Pattern:**
  - Pair with Firecrawl (discovery + extraction) for comprehensive research pipelines.

#### 2.4 **Brave Search**

- **Docs:**
  - API: [Brave Search API](https://brave.com/search/api/)
  - Comparison: [Firecrawl blog on Brave Search](https://www.firecrawl.dev/blog/best-deep-research-apis)
- **Capabilities:**
  - Privacy‑first, independent index.
  - Structured snippets and SERP data.
- **Use:**
  - High‑volume search, privacy‑sensitive apps, or index‑diversity‑driven research.

### 4. **Open‑source / framework‑level tooling**

While not “deep‑research APIs” per se, these are the building blocks for custom agents.

#### 4.1 **Open‑source research agent frameworks**

- **LangChain / LlamaIndex**
  - Position: glue between search APIs (Tavily, Serper, Exa, etc.) and LLMs.
  - Pattern:
    - Search → fetch → summarize → refine → finalize.
  - Example: a “Tavily + LangChain research agent” that iteratively expands queries.

- **Custom MCP‑style orchestrators**
  - OpenAI’s “deep research” now supports MCPs; you can build your own MCP‑compatible controllers that route to multiple APIs, files, and external tools.

#### 4.2 **AnyCap Deep Research (agent‑side CLI)**

- **Info:**
  - Article: [Deep Research APIs Compared 2026 – AnyCap](https://anycap.ai/page/en-US/ai/deep-research-apis-compared-2026)
- **How it works:**
  - CLI command: `anycap research --query "..."`
  - Decomposes query → runs multi‑round web search → crawls top sources → synthesizes structured Markdown with citations.
- **Use:**
  - Agent‑first research that fits into existing shell‑based pipelines.

### 7. **Emerging patterns and best practices (2026)**

#### 7.1 **Architectural patterns**

- **Agentic planning + execution:**
  - Use Gemini Deep Research or custom agents to plan multi‑step research before executing.
- **Multi‑tool pipelines:**
  - Example:
    Tavily/Exa → Firecrawl → Gemini → human review.
- **File‑search + web search:**
  - Gemini and OpenAI both support “search my files + web” for hybrid‑source research.

#### 7.2 **Skills and “research‑move” taxonomies**

- **Move 1 – Scoping:**
  - Translate ambiguous questions into research questions.
- **Move 2 – Query decomposition:**
  - Break “What should we do about X?” into smaller sub‑questions.
- **Move 3 – Iterative probing:**
  - Use synthesizers to generate follow‑up questions.
- **Move 4 – Evidence‑grading:**
  - Assign reliability scores to sources and flag weak evidence.
- **Move 5 – Contrastive synthesis:**
  - “Here are 3 conflicting views; reconcile them.”

#### 7.3 **Best practices**

- **For API design:**
  - Return citations with link, domain, date, and a short reason‑tag.
  - Support schema‑driven outputs for downstream pipelines.
- **For prompts:**
  - Force structure: “sections: background, competitors, risks, recommendations”.
  - Request “limitations” and “what’s missing” explicitly.
- **For reliability:**
  - Cross‑check high‑impact claims against at least two independent sources.
  - Track “claims → sources → confidence level” in a structured log.
