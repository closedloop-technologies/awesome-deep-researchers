# Gemini grounded Result

This comprehensive technical brief covers the state of **Agentic Deep Research** in 2026. Deep research systems have shifted from basic, single-shot search-augmented retrieval (RAG) to autonomous, stateful, multi-step reasoning agents that write code, navigate secure browser environments, execute nested planning loops, and recursively self-correct.

---

## 1. Paradigm Shifts & Core Concepts in 2026

Deep research in 2026 is defined by several core architectural shifts:

*   **Search as Code (SaC):** Pioneered by Perplexity in mid-2026, SaC moves away from treating search as a basic black-box tool call. Instead, the agent autonomously writes and executes Python/JS code to interact directly with search engine APIs. This enables dynamic sorting, custom pagination, and post-filtering before processing pages.
*   **Hosted Sandboxed Runtimes:** The "run-on-my-laptop" model has hit security and timeout limitations for production. Workloads have transitioned to hosted, isolated Linux cloud sandboxes (such as Anthropic Managed Agents and OpenAI Responses API) where agent sessions can run uninterrupted for hours.
*   **Two-Stage Architectures (Agent-to-Workflow):** Leading systems separate *exploration* from *synthesis*. Exploratory research is handled by autonomous stateful loops (high-degree agentic pathfinding). Once facts are gathered, the final report is generated through a deterministic, tightly constrained writing workflow (Evaluator-Optimizer pattern) to prevent AI "slop".

### OpenAI Responses API (`o3-deep-research` & `o4-mini-deep-research`)
OpenAI exposes its native deep research capabilities programmatically through the **Responses API**.
*   **Models:**
    *   `o3-deep-research`: The premier reasoning model optimized for legal, scientific, and massive data synthesis. ($10/M input, $40/M output; 200k context window).
    *   `o4-mini-deep-research`: Extremely fast, cost-effective alternative ($2/M input, $8/M output).
*   **Key Features:** Must be initialized with at least one data source (`web_search_preview`, remote Model Context Protocol (MCP) servers, or custom `file_search` vector stores). Supports long running actions via `background: True` and webhooks.

### Perplexity Agent API
A robust developer surface designed specifically for agentic search.
*   **Search as Code (SaC):** Allows agents to programmatically query and filter Perplexity's index.
*   **Finance Search:** Combines licensed financial data, SEC transcripts, earnings estimates, and real-time market data directly inside a unified tool call, bypassing custom scraper logic.
*   **Pricing Pattern:** Billed per-invocation of tools ($0.005 per `web_search`, $0.0005 per `fetch_url`) alongside base token rates. Fully compatible with OpenAI client SDK schemas.

### Anthropic Claude Managed Agents
Launched in April 2026, this is a fully hosted runtime execution platform.
*   **The Architecture:** Instead of exposing a simple chat endpoint, Anthropic provisions a secure cloud-hosted Linux sandbox for Claude.
*   **Key Features:** Equipped with 8 built-in tools (bash, read, write, edit, glob, grep, web_fetch, web_search). Sessions run for up to 3 hours.
*   **Vault-Scoped Credentials:** Allows developers to securely inject client tokens (e.g., Stripe, Slack, GitHub keys) into the hosted environment using vault proxies, meaning the credentials remain completely hidden from the underlying model.
*   **Pricing:** Standard Claude token rates + $0.08 per session-hour.

## 3. Top Open-Source Frameworks & Repositories

An open-source pipeline for long-horizon deep research trajectory synthesis.

*   **Performance:** Achieves **54.8% on BrowseComp-Plus**, outperforming Claude Opus, Gemini 2.5 Pro, and DeepSeek-R1.

### 2. [OpenClaw](https://github.com/openclaw/openclaw)
The viral breakout star of 2026 (surging past 210,000+ GitHub stars).
*   An entirely local, on-device agent platform that runs offline-capable LLMs.
*   Connects to 50+ communication integrations (iMessage, Signal, Slack, WhatsApp).
*   Operates with extreme autonomy; it is capable of running bash commands locally and **writing its own skill extensions** dynamically to solve unexpected research bottlenecks.

### 3. [open_deep_research](https://github.com/langchain-ai/open_deep_research) (LangChain-AI)
A stateful, graph-based deep research agent built natively on **LangGraph**.
*   Uses graph states to manage planning, parallel querying, deduplication, and compilation.
*   Natively compatible with LangGraph Studio for visual tracing, local human-in-the-loop editing, and MCP server tools.

### 4. [gpt-researcher](https://github.com/assafelovic/gpt-researcher) (assafelovic)
*   The most mature, community-vetted Python-based research framework.
*   Relies deeply on the **Model Context Protocol (MCP)** to isolate scraping and search tools into separate, secure microservices.

## 5. Tips, Tricks & Orchestration Best Practices

1.  **The "Writer-Reviewer" Split (Avoid Agentic Slop):** Do not let your exploratory search agent write the final report. Instead, save the research artifacts into a unified JSON/Markdown schema (the "research dump"). Pass this dump to a non-agentic LLM executing a highly constrained **Evaluator-Optimizer** loop with explicit writing stylesheets (such as prohibiting generic summaries, ensuring exact active-voice syntax, and enforcing strict citation rules).
2.  **Web Scrapers vs. Browser Use:** Only use full browser-automation frameworks (like `browser-use`) when interacting with behind-the-firewall web applications or complex UI states. For normal research, prioritize clean extraction APIs (like **Firecrawl**, **Jina Reader**, or **JigsawStack**). This saves up to 90% in execution speed and prevents brittle UI selector breaks.
3.  **MCP-First Integrations:** When writing custom tool functions for your agents, package them as **Model Context Protocol (MCP)** servers. This separates tool execution environments from the main orchestrator, making it trivial to port tools (such as database lookups, secure scrapers, or internal code engines) between different agent frameworks.
4.  **Enforce Visual Evidence Fidelity (VEF):** When scraping research papers with charts and data tables, do not rely on raw OCR or simple PDF-to-text parsers. Use deep parsing tools like **RAGFlow** to maintain document layout context, and pass tables as structured markdown or image crops directly to multimodal vision-enabled reasoners to prevent citation hallucinations.
